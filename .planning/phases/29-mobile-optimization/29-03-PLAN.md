### DealTypeTabs.tsx — Embla 스와이프 추가

**현재 구조 파악 포인트:**
- 어떤 state/URL 파라미터로 현재 탭을 관리하는지 (nuqs? useState?)
- 탭 버튼 행과 탭 콘텐츠가 어떻게 구성되어 있는지
- 어떤 탭 items가 있는지 (거래내역/시설/관리비/AI코멘트 등)

**Embla 추가 패턴 (RESEARCH.md Pattern 5 + Pitfall 6):**

```tsx
import useEmblaCarousel from 'embla-carousel-react'
import { useCallback, useEffect } from 'react'

// 컴포넌트 내부:
// activeTabIndex: 현재 탭의 배열 인덱스 (URL 상태 또는 prop에서 도출)
const [emblaRef, emblaApi] = useEmblaCarousel({
  startIndex: activeTabIndex,  // URL 직접 진입 시 올바른 탭으로 시작
})

// 스와이프 완료 → URL 상태 동기화 (Pitfall 6 해결)
useEffect(() => {
  if (!emblaApi) return
  const onSelect = () => {
    const newIndex = emblaApi.selectedScrollSnap()
    // 현재 사용 중인 setTab/setActiveTab 함수 호출 (nuqs 또는 router.push)
    setActiveTabByIndex(newIndex)  // 실제 함수명으로 교체
  }
  emblaApi.on('select', onSelect)
  return () => { emblaApi.off('select', onSelect) }
}, [emblaApi])

// URL 탭 인덱스 변경 시 Embla 동기화 (탭 버튼 클릭 시)
useEffect(() => {
  if (emblaApi && emblaApi.selectedScrollSnap() !== activeTabIndex) {
    emblaApi.scrollTo(activeTabIndex)
  }
}, [emblaApi, activeTabIndex])
```

**콘텐츠 영역 Embla 래핑:**
```tsx
{/* Embla 컨테이너 — overflow-hidden 필수 */}
<div ref={emblaRef} className="overflow-hidden">
  {/* Embla 트랙 — flex 레이아웃 */}
  <div className="flex">
    {tabs.map((tab) => (
      <div key={tab.key} className="min-w-full">
        {/* 탭 콘텐츠 */}
        {tab.content}
      </div>
    ))}
  </div>
</div>
```

**탭 버튼 행 — 44px 터치 타겟 보장 (D-10):**
기존 탭 버튼이 `minHeight: 32` 이면 `min-h-[44px]` 로 업그레이드:
```tsx
// Before: style={{ minHeight: 32 }}
// After: className="min-h-[44px]" (기존 스타일과 병합)
```

- DealTypeTabs.tsx는 이미 `'use client'` — 그대로 유지
- Embla는 content 영역만 감싼다. 탭 버튼 행은 Embla 외부에 유지.
- 탭 버튼 클릭 시: `emblaApi.scrollTo(index)` 호출하여 Embla와 동기화
