  <action>
아래 SQL을 Supabase MCP(`execute_sql` 계열 도구, 사용 가능한 경우 우선) 또는 `psql "$SUPABASE_DB_URL"`(OPS-01 백업 워크플로와 동일한 Direct connection URI 사용)로 실행하여 전체 DB 용량과 주요 테이블별 용량(인덱스 포함)을 측정한다:

```sql
-- 전체 DB 용량 (인덱스·TOAST 포함, 무료 티어 500MB 한도와 직접 비교 가능한 수치)
select pg_size_pretty(pg_database_size(current_database())) as total_db_size;

-- 주요 테이블별 용량 (테이블+인덱스+TOAST) — 큰 순서대로
select
  relname as table_name,
  pg_size_pretty(pg_total_relation_size(relid)) as total_size,
  pg_size_pretty(pg_relation_size(relid)) as table_only_size,
  pg_size_pretty(pg_indexes_size(relid)) as index_size
from pg_catalog.pg_statio_user_tables
order by pg_total_relation_size(relid) desc
limit 10;

-- 지역별 데이터 분포 참고 (경남 확장 기여분 추정)
select sgg_code, count(*) as tx_count
from transactions
group by sgg_code
order by tx_count desc;
```

측정 결과를 다음 임계값과 비교한다:
- < 400MB: 무료 티어 유지 가능, 여유 있음
- 400~500MB: 무료 티어 근접, 다음 확장(34+ phase) 전 Pro 전환 검토 필요
- > 500MB: 무료 티어 초과 확실, Pro 전환 필수

Supabase 대시보드(Settings → Billing/Usage)에서도 동일 수치를 교차 확인할 수 있다.
