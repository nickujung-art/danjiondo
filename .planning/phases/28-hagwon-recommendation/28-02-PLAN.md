<!-- NEIS acaInsTiInfo API 응답 구조 (RESEARCH.md Pattern 1에서 검증됨) -->
```typescript
// URL: https://open.neis.go.kr/hub/acaInsTiInfo
// 인증: KEY (query param) — NEIS_API_KEY env var
// 교육청 코드: ATPT_OFCDC_SC_CODE = 'S10' (경남)
// 페이지네이션: pIndex (1-based), pSize (최대 1000)

// 응답 구조:
// json.acaInsTiInfo[0].head[0].list_total_count  → 총 건수
// json.acaInsTiInfo[0].head[1].RESULT.CODE       → 'INFO-000' | 'INFO-200' | 'ERROR-*'
// json.acaInsTiInfo[1].row                        → 데이터 배열

interface NeisAcaRow {
  ACA_NM:           string  // 학원명
  ADMST_ZONE_NM:    string  // '창원시 성산구 내동' 등
  ACA_ASNUM:        string  // 학원지정번호 (upsert 키)
  REALM_SC_NM:      string  // '학교교과' | '예능' | '체능' 등
  LE_ORD_NM:        string  // 교습계열명
  LE_CRSE_NM:       string  // 교습과정명
  PSNBY_THCC_CNTNT: string  // 수강료 (자유형식, nullable)
  THCC_OTHBC_YN:    string  // 'Y' | 'N'
  FA_RDNMA:         string  // 도로명주소
  FA_RDNDA:         string  // 도로명상세주소
  FA_RDNZC:         string  // 우편번호
  FA_TELNO:         string  // 전화번호
  TOFOR_SMTOT:      number  // 정원
  REG_STTUS_NM:     string  // '등록' = 활성, 나머지 = 폐원
  ESTBL_YMD:        string  // 'YYYYMMDD'
  LOAD_DTM:         string  // 수정일자
}
```

`scripts/fetch-hagwon-neis.ts` 파일을 생성한다. 아래 사항을 구현한다:

**환경변수**: dotenv.config({ path: '.env.local' }). NEIS_API_KEY, NEXT_PUBLIC_SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY 필수.

**CLI 옵션**:
- `--dry-run`: DB upsert 없이 첫 페이지 건수만 출력
- `--zone=창원시`: 특정 구역만 실행 (기본: 창원시 + 김해시 모두)

**수집 대상**: `ADMST_ZONE_NAMES = ['창원시', '김해시']` 각각에 대해 순서대로 실행.

**API 호출 함수** `fetchPage(zoneName, pIndex)`:
```typescript
const url = new URL('https://open.neis.go.kr/hub/acaInsTiInfo')
url.searchParams.set('KEY', process.env.NEIS_API_KEY!)
url.searchParams.set('Type', 'json')
url.searchParams.set('ATPT_OFCDC_SC_CODE', 'S10')
url.searchParams.set('ADMST_ZONE_NM', zoneName)
url.searchParams.set('pIndex', String(pIndex))
url.searchParams.set('pSize', '1000')
```

**응답 파싱** (Pitfall 1, 2 방어):
- `json.acaInsTiInfo[0].head[0].list_total_count` → 총 건수
- `json.acaInsTiInfo[0].head[1].RESULT.CODE`:
  - `'INFO-000'` → 정상, row 파싱
  - `'INFO-200'` → 데이터 없음, 빈 배열 반환 (에러 아님!)
  - `CODE.startsWith('ERROR')` → throw new Error
- 총페이지 = Math.ceil(totalCount / 1000), 루프로 전부 수집

**수강료 파싱** `parseFeeAmount(text: string): number | null` (Pitfall 3 방어):
```typescript
const m = text.replace(/[,원]/g, '').match(/\d+/)
return m ? parseInt(m[0], 10) : null
```

**활성 필터**: `row.REG_STTUS_NM === '등록'` → is_active=true, 그 외 is_active=false

**개설일자 파싱**: ESTBL_YMD 'YYYYMMDD' → `${y}-${m}-${d}` ISO date string, 빈 문자열이면 null

**hagwon_db upsert**:
```typescript
await supabase.from('hagwon_db').upsert(rows, {
  onConflict: 'aca_asnum',
  ignoreDuplicates: false,
})
```

**upsert 배치 크기**: 100건씩 분할 (Supabase upsert 권장 한도)

**딜레이**: 페이지 호출 간 500ms sleep (API 부하 방지)

**진행 출력**:
```
[창원시] page 1/5 → 1000건 수집
[창원시] page 2/5 → 1000건 수집
[창원시] 완료: 4823건 upserted
[김해시] ...
```

**CLI 옵션**:
- `--missing-only`: location IS NULL인 행만 처리 (기본 동작, 권장)
- `--all`: location이 있어도 재처리
- `--dry-run`: geocoding 결과 출력만, DB 저장 없음
- `--limit=1000`: 처리 최대 건수 (기본 1000, Pitfall 4 Kakao 일 100,000회 한도 방어)

**hagwon_db에서 대상 행 조회**:
```typescript
let query = supabase.from('hagwon_db').select('id, name, address, address_detail')
if (missingOnly) query = query.is('location', null)
query = query.limit(limit)
const { data: rows } = await query
```

**Kakao 주소 검색 API 호출** `geocodeAddress(address: string)`:
```typescript
const url = new URL('https://dapi.kakao.com/v2/local/search/address.json')
url.searchParams.set('query', address)
const res = await fetch(url.toString(), {
  headers: { Authorization: `KakaoAK ${process.env.KAKAO_REST_API_KEY!}` },
})
const json = await res.json()
const doc = json.documents?.[0]
if (!doc) return null
return { lat: parseFloat(doc.y), lng: parseFloat(doc.x) }
```

**폐원 확인 생략**: is_active 플래그는 fetch-hagwon-neis.ts에서 REG_STTUS_NM='등록' 기준으로 이미 설정됨. 이 스크립트는 위경도 좌표만 채운다.

**hagwon_db UPDATE**:
```typescript
const point = `POINT(${lng} ${lat})`  // PostGIS: lng(경도) 먼저!
await supabase.from('hagwon_db')
  .update({ location: `SRID=4326;${point}` })
  .eq('id', row.id)
```

Note: Supabase PostGIS에서 geometry 컬럼에 WKT 직접 upsert 시 `SRID=4326;POINT(lng lat)` 형식 사용.
(ST_GeomFromText 사용 불필요 — Supabase client가 자동 변환)

**딜레이**: 요청 간 100ms sleep (Kakao 일 100,000회 한도 기준, 15,000건 처리 시 3시간 예상)

**진행 출력**: 100건마다 `[진행] N/M 완료 (geocoded: K, not_found: L)`

**not_found 처리**: geocoding 실패(빈 documents) → location null 유지, 경고 로그만
