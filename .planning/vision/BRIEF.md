## 한 줄 정의

**창원·김해 사람들이 가장 먼저 읽는 부동산 미디어**

---

## 1. 문제 — 지금 단지온도는 뭐가 문제인가

- 호갱노노 따라 기능만 붙인 "기능 집합체"가 돼버렸다
- 누구를 위한 서비스인지 불명확
- 전국 서비스 대비 기능 부족, 카페 대비 깊이 부족
- **집 살 생각 없으면 들어올 이유가 없다**

---

## 2. 방향 — 왜 로컬 미디어인가

| 자산 | 의미 |
|------|------|
| 창원부동산이야기 카페 **20만 회원** | 이미 독자가 있다 |
| 창원·김해 다루는 로컬 미디어 **경쟁자 0** | 선점 가능 |
| 호갱노노가 절대 못 만드는 **우리 동네 이야기** | 차별화 |

→ 데이터 기능은 콘텐츠 안에 녹인다. 기능 페이지가 아니라 기사 페이지로.

---

## 3. 브랜드 — "창부레터"

- **어원**: 창원부동산이야기 + 레터(뉴스레터)
- **도메인**: changbuletter.com
- **레터**라는 이름이 이메일 구독을 자연스럽게 암시
- 카페 20만 회원 → 뉴스레터 독자로 전환하는 흐름

> **제1 옵션 확정** — 이 이름 기준으로 진행

---

## 4. 타겟

| 우선순위 | 대상 | 특징 |
|---------|------|------|
| **MAIN** | 창원·김해 거주 30~50대 | 이미 내 집 있음, 카페 회원, 동네 소식 소비 |
| **MAIN** | 이 지역 처음 오는 사람 | 삼성·포스코 임직원 이전, 학군·교통 정보 필요 |
| SUB | 로컬 투자자 | 갭투자·재건축, 데이터 기반 |

---

## 5. 콘텐츠 4종

| 콘텐츠 | 주기 | 채널 | 역할 |
|--------|------|------|------|
| 주간 실거래가 카드뉴스 | 주 1회 | 인스타 + 카페 | 바이럴, 이미 자동화됨 |
| 기획기사 (동네 분석) | 월 2~4회 | 사이트 + 카페 | 구글 유입·신뢰도 핵심 |
| 랭킹 인포그래픽 | 주 1회 | 인스타 스토리 | 거래량·상승률 TOP 5 자동 생성 |
| 지역 가이드 | 월 1~2회 | 사이트 | 이전자 수요, 롱테일 SEO |

---

## 6. 사이트 구조 변경

```
기존: 단지 검색 중심
변경: 콘텐츠 피드 중심 홈 + 콘텐츠 섹션 신설

신규: 홈(재설계), 콘텐츠, 지역 가이드
유지: 단지 검색, 마이페이지
depth↓: 지도 (메인 네비에서 빠짐)
```

---

## 7. 유저 동선 (4단계 퍼널)

```
진입 → 랜딩 → 탐색 → 전환/락인

[인스타·카페] → 콘텐츠 상세(★ 주요 랜딩)
[구글 검색]   → 지역 가이드
[이메일]      → 콘텐츠 상세 (재방문)

락인 포인트:
  즉시 투표 참여 → 커뮤니티
  북마크 저장    → 소셜 로그인 → 회원가입
  뉴스레터 구독  → ★ 핵심 전환
```

**콘텐츠 상세 페이지가 가장 중요한 랜딩 페이지**  
인스타 링크 클릭 → 콘텐츠 상세 → 구독 CTA가 기본 퍼널.

## 8. 다음 단계 (우선순위)

1. **콘텐츠 상세 페이지** (최우선) — 카드뉴스 풀뷰 + 단지 데이터 + 구독 CTA
2. **홈 피드형 재설계** — 검색창 → 콘텐츠 피드, 네비에 '콘텐츠' 추가
3. **이메일 뉴스레터 발송** — Resend API, 주간 큐레이션 이메일 → 창부레터 브랜드 완성
4. **슬라이드 카드뷰어 + 커뮤니티** — 체류시간·재방문율 핵심
5. **지역 가이드 신설** — 경쟁자 없는 SEO 자산

## 10. 콘텐츠 상세 페이지 기획 (2026-07-01)

**URL**: `/content/[slug]`

### 페이지 구성 (위 → 아래)

| 영역 | 내용 |
|------|------|
| 헤더 | 태그, 제목, 날짜 |
| 메인 콘텐츠 | 타입별 분기 |
| 관련 단지 | 카드 여러 개 (다대다) |
| VS 투표 | 로그인 필수, 즉시 결과 공개 |
| 뉴스레터 CTA | 구독 유도 |
| 다음 콘텐츠 | 추천 2~3개 |

### 콘텐츠 타입 2종

- **카드뉴스** → HTML 슬라이드 뷰어 (PNG 아닌 HTML 렌더링, SEO 대응)
- **기획기사** → Tiptap 블록 에디터, JSON 저장 (텍스트 + 이미지 블록)

### VS 투표

- 어드민에서 질문 제목 / 왼쪽 옵션 / 오른쪽 옵션 직접 입력
- 로그인 후 투표 → 즉시 퍼센트 바 공개
- 락인 포인트 역할 (비로그인 → 로그인 유도)

### DB 구조

```
contents          — id, slug, type(card_news|article), title, tag, body(JSON), published_at
                    vote_question, vote_left, vote_right
content_complexes — content_id ↔ complex_id (다대다)
content_votes     — content_id, user_id, choice(left|right)
```

### 미결 (다음 세션)

| 항목 | 옵션 |
|------|------|
| 랭킹 인포그래픽 타입 | 카드뉴스 타입에 합치기 vs 별도 타입 신설 |
| 공유 기능 | 카카오톡·링크 복사 버튼 여부 |
| OG 이미지 | 카드뉴스 첫 슬라이드를 SNS 썸네일로 쓸지 |

## 9. 결정 사항 (2026-07-01 확정)

| 질문 | 결정 |
|------|------|
| 단지온도와의 관계 | **완전 리브랜딩** — 싹 다 새로 고침 |
| "창부레터" 이름 | **제1 옵션 확정** (가안 → 진행 기준) |
| 카페 vs 창부레터 역할 분담 | **카페 운영진 = 창부레터 운영자 (동일인)** — 카페의 공식 미디어 확장판으로 운영. 카페=커뮤니티, 창부레터=미디어 |
| 기획기사는 누가 쓰나 | **직접 + AI 보조** — 월 2~4회 |
